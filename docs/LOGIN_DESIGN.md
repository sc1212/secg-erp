## 6. SESSION & SECURITY

```
LOGIN SECURITY:
  - JWT access token: 15 minute expiry
  - Refresh token: 7-day expiry, httpOnly secure cookie
  - Refresh token rotation (old token invalidated on refresh)
  - "Keep me signed in" extends refresh to 30 days
  - Max 5 active sessions per user
  - Bcrypt password hashing (12 rounds)
  - Rate limiting: 5 failed attempts â†’ 15 minute lockout
  - Suspicious login alerts (new device/location)

PASSWORD REQUIREMENTS:
  - Minimum 8 characters
  - Must contain uppercase and lowercase
  - Must contain at least one number
  - Special character recommended (not required)

FUTURE (Phase 2):
  - MFA via authenticator app (TOTP)
  - SSO/SAML for Enterprise tier
  - Magic link login option
  - Session management page (view/revoke active sessions)
```
